Enabling Unicode

Do not enable field-selectable Unicode until you have reviewed the risks and limitations associated with this feature. Do not enable Unicode if your database contains multi-byte data. Converting existing multi-byte data fields to Unicode may result in a loss or corruption of data.

Tell me more...

When field-selectable Unicode is enabled:

  • Unicode field types (Unicode String and Unicode Memo) become available in the Database Manager and Application Architect Entity Designer. Unicode String displays Unicode text on the clients.
  • GroupTranslator sets conditions to UTF-16.
  • Saleslogix uses a new code page for ANSI String and UTF16.
  • The System Summary Report is updated to indicate Unicode is enabled.
  • Database storage requirements may increase. Ensure your main office and Remote user computers have sufficient hardware to store and process Unicode data.

Field-selectable Unicode is limited to specific feature areas. Extending the database for Unicode requires a strong knowledge of Saleslogix and its database schema. Before implementing this feature, you should carefully review Understanding Unicode for details. Persons implementing this feature assume full responsibility and understand the risks and limitations associated with Unicode in Saleslogix. Saleslogix strongly recommends you back up your database before making any Unicode changes.

If you convert an existing field to Unicode or create a new Unicode field and write data to your database, you cannot revert your installation to ANSI data types. You can enable/disable Unicode and create/remove new Unicode fields only until you convert data or write Unicode data to your database. Once Unicode data has been written to the database, you will corrupt your data if you disable the changes.

To enable

  1. Use Administrator to do the following:
  1. On the Tools menu, click Options.
  2. Click the Database tab.
  3. Select the Allow Unicode field types check box.
  4. Read the warning message, and then click Yes.
  5. Click OK.
  1. (Microsoft SQL Server Only) Disable auto-translate on your database server.
    1. Open the Connection Manager.
    2. In the SalesLogix Connection Manager dialog box, select your database connection and click Edit.
    3. Click the All tab.
    4. Select the Auto Translate property and click Edit Value.
    5. In the Property Value box, select False.
    6. Click OK.
    7. Save your settings and exit the Connection Manager.
    8. Restart the server.
  2. (Oracle Only) If you are creating a new database instance, use AL32UTF8 as the Normal character set.
  3. (Windows 2003/XP only) Configure supplemental language support on Web Host and Web Client machines if they must support East Asian or other multi-byte character sets:
    1. In the Control Panel, open Regional and Language Options.
    2. On the Regional Options and Advanced tabs, verify the correct language is selected in the drop-down list.
    3. If your required language is not available in the list, click the Languages tab and select the boxes for Supplemental language support. For example, if Chinese, Japanese, or Korean languages will be used or stored in the database, check Install files for East Asian languages. A file copy and restart may be required. When finished, select the appropriate language on the Regional Options and Advanced tabs and restart.
  4. If you anticipate that your database will continue to contain some multi-byte data, then verify that the Language for non-Unicode programs value is defined on the Web Host and Web Client machines. This setting is used to determine the code page that is used to convert between multi-byte string data and Unicode data.
    1. In the Control Panel, open Regional and Language Options.
    2. On the Advanced tab (Windows XP/2003) or the Administrative tab (Windows 2008/7), select the appropriate language.
  5. (Microsoft Office 2007 only) Enable UTF-8 support for Internet Protocols in Microsoft Outlook. UTF-8 support must be enabled when users are exporting extended characters to e-mail.
    1. Open Microsoft Outlook.
    2. On the Tools menu, click Options.
    3. Click the Mail Format tab.
    4. Click International Options.
    5. Under Internet Protocols, select Enable UTF-8 support for mailto: protocol.
    6. Click OK.
